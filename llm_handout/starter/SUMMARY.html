2,000-Step LLM Speedrun - Summary

Bits-per-byte (bpb) minimization under a 2,000-optimizer-step / 2,000,000-parameter cap, CPU-only, pure PyTorch. See RUNLOG.md for the full run-by-run trail and NOTES.md for the condensed writeup.

Final submitted checkpoint.
Best config: RoPE + Muon + WSD schedule + batch 64, stacked on top of the BPE-tokenizer base - dev bpb 1.6777 (baseline was 2.3718, -29.3% relative).

  • Tokenizer: from-scratch byte-level BPE (vocab 2048), trained only on train_corpus.txt, Unicode-category-aware pre-tokenization (fixes Devanagari matra/virama fragmentation that plain \w regex splitting causes), byte-level fallback base vocab - structurally lossless on arbitrary UTF-8 by construction.
  • Positional encoding: RoPE instead of a learned absolute position table - better bpb and 20,480 fewer params (Run 8).
  • Optimizer: Muon (Newton-Schulz orthogonalized updates) on the hidden 2D weights, AdamW on everything else - the single biggest individual win of the project (Run 10).
  • Schedule: WSD (warmup-stable-decay) - beat cosine decay by holding LR high longer before a sharp final decay (Run 9).
  • Batch size: 64 (up from the baseline's 8) - the step budget is the binding constraint, not compute, so more distinct tokens per step matters more than gradient-noise reduction. Batch 32 won first (Run 6); batch 64 was re-tested on top of the full stack and won again (~4.8 epochs of the corpus, within Muennighoff's safe repeated-data range).
  • Architecture: unchanged 4-layer/4-head/160-dim transformer, untied input/output embeddings (weight tying was tested twice, at two vocab sizes, and rejected both times - see RUNLOG Runs 3 & 5), flat N(0, 0.05) init (GPT-2/muP-style scaled init was tested and rejected - see Run 2).
  • Stacking: RoPE, Muon, and WSD were each validated individually first, then combined - the combination beat every individual result cleanly. Batch 64 was then re-tested on top of that combined stack and won again.

Human vs. machine work split

The coding agent wrote all of the code. The literature review and the direction of every change - what to try, in what order, and what to do with the results - came from the human.

  • Agent (Claude Code): all code - tokenizer, model, training loop, optimizers (including Muon/NorMuon) - plus running and scoring every training run and writing RUNLOG.md and this document.
  • Human: literature review and primary-source verification for every externally-suggested technique, and direction on what to change, in what order, and what to do with each result (including the call to kill and re-sequence the oversubscribed parallel runs, and which conditional experiments to run given the deadline).
